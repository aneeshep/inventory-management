Factory Inventory Management System

System Architecture & Technical Reference

Vue 3 FastAPI In-Memory Data No Database

Tech Stack

Frontend

Vue 3 + Vite

Single-page application on port 3000. Composition API throughout, with reactive composables for shared filter and auth state.

Vue 3.4 Vue Router 4.3 Axios 1.6 Vite 5.2
Backend

Python FastAPI

REST API on port 8001. All filtering and aggregation happens in-process. Pydantic validates every response. CORS open for local dev.

FastAPI 0.110 Uvicorn 0.24 Pydantic 2.5 uv
Data Layer

JSON + In-Memory

Seven JSON files loaded at server startup into Python lists. No database — changes made at runtime are lost on restart.

inventory.json orders.json spending.json backlog_items.json +3 more
Internationalisation

EN / JA Locales

Full English and Japanese support via a custom useI18n composable. Locale persisted in localStorage. Currency auto-converts (USD ↔ JPY at 150).

en.js ja.js localStorage

API Endpoints

Method Path Query Filters Description
GET /api/inventory warehouse, category All stock items with current quantities
GET /api/inventory/{id} — Single inventory item by ID
GET /api/orders warehouse, category, status, month Order list; month accepts YYYY-MM or Q1-2025
GET /api/orders/{id} — Single order by ID
GET /api/dashboard/summary warehouse, category, status, month Computed KPIs: inventory value, low-stock count, pending orders, backlog total
GET /api/demand — Demand forecasts with trend indicators (increasing / stable / decreasing)
GET /api/backlog — Backlog items enriched with has_purchase_order flag at runtime
GET /api/spending/summary — Procurement, operational, labor, overhead totals + YoY % change
GET /api/spending/monthly — Month-by-month cost breakdown (Jan–Dec)
GET /api/spending/categories — Spending split by product category
GET /api/spending/transactions — Recent transaction list
GET /api/reports/quarterly — Q1–Q4 stats: revenue, order count, fulfillment rate
GET /api/reports/monthly-trends — Month-by-month revenue and volume trends
GET /api/purchase-orders/backlog/{id} — Purchase order linked to a specific backlog item
POST /api/purchase-orders — Create a purchase order for a backlog item

All, Delivered, Shipped, Processing, Backordered

Filters with value "all" are omitted from query params. All filter state is managed globally by the useFilters() singleton composable and watched by each view.
